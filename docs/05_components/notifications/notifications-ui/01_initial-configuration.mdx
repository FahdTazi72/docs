## Prerequisites

Make sure the [back end](../../notifications-server/notify-configuring-runtime) has been configured correctly before you install the front-end components. 

Also, note that almost every section of each component requires a permission to perform the specific action related to that section; check the [permissions](../notify-permissions) available and make sure your user has the permission needed for the section you want to implement or test. 

You can read more about permissions and rights in the [authorisation](../../../../server/access-control/authorisation#users-profiles-and-right-codes) section.

## Setting up in an existing application

If you want to install in an existing application, you have a choice. You can install all the components in one simple command, or you can manually install only the specific components that you want.

### Add all components at once

Run this command (use the name of your app folder instead of _app-folder-name_):
```shell
npx -y @genesislcap/genx@latest add app-folder-name -x -s genesislcap/pbc-notify-seed
```

After this, do a `npm run bootstrap` to install everything.

### Manual installation 

The manual method enables you to add each component individually to an existing application.

In the **client/package.json** file, add this in `"dependencies"` section:

```json
"@genesislcap/pbc-notify-ui": "~1"
```

Run `npm run bootstrap` to install it.

:::important
Always check the latest version available at [genesisnpm](https://www.npmjs.com/~genesisnpm).
:::

#### Foundation Notification Dashboard

Add the component to the **src/components/component.ts** file:

```typescript
...
FoundationNotificationDashboard;
...
```

In the **src/routes/config.ts** file, add the route for the component:

```typescript
{
  path: 'notification-dashboard',
  element: async () => (await import('@genesislcap/pbc-notify-ui')).FoundationNotificationDashboard,
  title: 'Notification Dashboard',
  name: 'notification-dashboard',
  settings: { autoAuth: true },
  navItems: [
    {
      title: 'Notification Dashboard',
      icon: {
        name: 'bell',
        variant: 'solid',
      },
    },
  ],
},
```

This creates a single-page route [Notification Dashboard](/../../notifications-ui/notify-ui-components#notification-dashboard) component.

Again, check the user permissions, because each tab of the dashboard is ruled by a permission (check the available [list of permissions](../notify-permissions)). The user only sees the sections for which they have permission.

#### Foundation Inbox Flyout

Inbox Flyout is a ready-to-use component for [Inbox](/../../notifications-ui/notify-ui-components#inbox) functionality.

Add the component in the `src/components/component.ts` file:
```typescript
...
FoundationInboxFlyout;
...
```

Declare it in the **src/layouts/default.ts*8 file. Here's an example:

```html
<div class="container">
  <foundation-header
    show-notification-button
    @notification-icon-clicked="${x => x.inbox.open()}"
    :routeNavItems="${(x) => x.config.getNavItems()}"
  >
  </foundation-header>

  <foundation-inbox-flyout ${ref('inbox')}></foundation-inbox-flyout>

  <zero-notification-listener resource-name="ALL_NOTIFY_ALERT_RECORDS">
    <div class="content">
      <slot></slot>
    </div>
  </zero-notification-listener>
</div>
```

:::important
It's crucial to import ```zero-notification-listener``` too, a foundation-ui component responsible for showing screen alerts to the user.
:::

If your main `container` div is typed as `ClientAppRouter`, then consider adding the following to avoid errors:

```typescript
type ClientAppRouter = FoundationRouter & {
  store: Store,
  inbox: FoundationInboxFlyout
};
```

#### Foundation Inbox

You can put an Inbox component that anywhere you want: on a page div, modal, dialog, etc. 

Add the component to the **src/components/component.ts** file:
```typescript
...
FoundationInbox;
...
```

When you generate a blank project, it puts the Inbox inside a flyout using the foundation-store to control its open-closed state:

```html
<zero-flyout
  position="right"
  @closed="${(x, c) => c.parent.$emit('change-inbox-display', false)}"
  ?closed="${() => !store.inboxDisplayState}"
  data-pbc-asset-id="inbox-flyout"
>
  <div slot="title">Alerts</div>
  <foundation-inbox></foundation-inbox>
</zero-flyout>
```

Check the [generated project with the pbc-notify-seed](#on-a-new-application) for more details.
